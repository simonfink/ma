in examples -> client_sub.py
add mqttc.username_pw_set(username="xxx", password="xxx")
connect to 172.18.252.25:1883, timeout (60) is not relevant
subscribe to topic "test/smartvalve"

HTTP
----
POST data to rabbitmq
body = '{"properties":{},"routing_key":"test.smartvalve","payload":"python http post test","payload_encoding":"string"}'
url = "/api/exchanges/%2f/amq.topic/publish" (and also 172.18.252.25:15672 -> web ui from rabbitmq), %2f is the default vhost, exchange could also be changed
the http frame gets split by tcp -> in wireshark: after tcp handshake another tcp frame -> is currently a http frame (only header). followed by the http body. psh flag in first frame -> push: the content length of the following body get send, so that server (or client) can reserve some space for that data
in python less overhead than in java (HttpURLConnection sends Pragma + value and Cache-Control + value, can't remove that)
